Introduction
01

Context & Problem

  • Tunisian news lives on separate, independent sites
  • Following it means opening each site by hand, every day
  • Nothing lets you search across sources
  • Nobody measures the tone of the coverage
  • Nobody maps which regions the news actually covers
4+ sitesindependent newsrooms
→
Fragmentedno shared index
→
Blind spotno search, no mood, no map

Project Objectives
03

Project Objectives

  • Automate news collection from Tunisian sources
  • Enrich every article: sentiment, theme, region, embedding
  • Enable search by meaning, not just keywords
  • Provide an assistant that answers from real articles
  • Run as a scheduled, monitored, reproducible pipeline

One Automated Pipeline

Orchestration
15

Orchestration & Tracking

Prefect: this morning's run, COMPLETED in 7m 26s
  • Prefect flow, scheduled 06:00 every day
  • HTTP trigger for on-demand runs
  • Retries: 2× / 60s on failure
  • One pipeline_metrics row per run: articles, topics, sentiment
  • MLflow logs every clustering run: params, metrics, model
16

Experiment Tracking

  • Every BERTopic clustering run is logged
  • Params: embedding model, min cluster size
  • Metrics: number of topics, outlier share
  • Latest run: 6 topics, 16.7% outliers
  • Runs comparable and reproducible over time
